# 📂 Database Study Notes

## 1. Database Basics
### RDBMS vs NoSQL
* **RDBMS (관계형 데이터베이스)**: 번호, 이름, 주소 등 구조화된 데이터를 테이블 형식으로 관리 (예: MariaDB, MySQL).
* **NoSQL**: SNS 데이터와 같이 고정된 구조가 없는 비정형 데이터 저장에 적합.

### MariaDB Access & Commands
MariaDB는 MySQL의 파생 프로그램으로 사용법이 유사합니다.
- **접속**: `mariadb -uroot -p` (비밀번호 입력)
- **명령어**: `SHOW DATABASES;` (반드시 세미콜론 `;` 포함)

> **이미지 저장**
> DB에 사진 파일을 직접 넣으면 용량이 무거워져 성능이 저하됩니다. 일반적으로 DB에는 **파일의 저장 경로(Path)**만 저장합니다.

| 구분 | 용어 1 | 용어 2 | 한국어 |
| :--- | :--- | :--- | :--- |

## 3. 정규화 (Normalization)
데이터 중복을 최소화하고 무결성을 유지하기 위해 스키마를 최소 단위로 쪼개는 과정입니다.

### 제1정규형 (1NF)
* **원자성 확보**: 모든 속성은 반드시 하나의 원자 값만 가져야 함.
* 한 칸(Cell)에는 하나의 데이터만 저장.

### 제2정규형 (2NF)
* **부분 함수적 종속성 제거**: 기본키가 복합키(Composite Primary Key)일 때, 키의 일부에만 종속되는 컬럼을 분리.
* 현재 테이블의 주 목적과 상관없는 정보는 별도 테이블로 이동.

### 제3정규형 (3NF)
* **이행적 함수 종속성 제거**: 기본키가 아닌 일반 컬럼끼리 종속되는 관계를 제거.
* 기본키와 직접 관계없는 데이터는 별도 테이블로 분리.

### BCNF (Boyce-Codd 정규형)
* 모든 결정자가 후보키(Candidate Key)가 되도록 설계.

## 4. SQL Classification
사용 용도에 따라 DDL, DML, DCL로 구분합니다.

| 분류 | 설명 | 주요 명령어 | 사용 주체 |
| :--- | :--- | :--- | :--- |
| **DDL** (정의어) | DB 구조 정의, 수정, 삭제 | `CREATE`, `ALTER`, `DROP` | DBA |
| **DML** (조작어) | 데이터 검색, 입력, 수정, 삭제 | `SELECT`, `INSERT`, `UPDATE`, `DELETE` (CRUD) | 개발자, 사용자 |
| **DCL** (제어어) | 권한 부여, 보안, 트랜잭션 | `COMMIT`, `ROLLBACK`, `GRANT`, `REVOKE` | DBA |

---

## 5. 정규화 예시 (Table Splitting)
직원 정보와 가족 정보를 하나의 테이블에 두지 않고 분리하여 관리합니다.

* **직원 테이블**: `사번(PK)`, `직원명`, `전화`, `부서`
* **가족 테이블**: `가족ID(PK)`, `사번(FK)`, `가족명`, `관계`, `성별`
